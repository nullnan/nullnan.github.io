修改配置

在解压源代码之后,我们为了方便管理用git init新建一个git仓库来管理配置

suckless开发的软件有个显著的特点就是需要通过修改头文件和打patch来启用修改功能.

同样,st的原生是不支持连字的,为了获得功能,我们必须在patches上面获得所需要的patch然后打上.下载diff文件后需要用

之后,如果有冲突,还需要手动解决.

通过足够多的patch和配置，我们就能获得我们想要的定制化终端，是不是很棒呢？

安装

stow安装

之前在安装emacs的时候用的stow现在可以派上用场，首先修改config.h 中的PREFIX在make install之后切换到目录执行下stow就行，这样的好处是方便快捷删除和切换版本（虽然st本来文件就不多．．．）

基于包管理器安装

替换系统安装的st

首先用sudo apt install stterm安装好官方版本st，之后用ln -s /usr.local/bin/st /usr/bin/st来替换掉官方的st．然而，可以预见的是，如果官方源更新了，那我们的替换就失效了．

然而我并不会打deb包，或许我可以研究一下deb或者nixpkg来打一个包．
